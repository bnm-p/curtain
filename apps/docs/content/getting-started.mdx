# Getting Started

## Prerequisites

- Next.js 13+ with **App Router**
- Node.js 18+ or Bun 1.0+
- [GSAP](https://gsap.com) 3.0+ (free tier is fine)

---

<Steps>

### Run the init command

In your Next.js project root, run:

```sh
npx view-motion init
```

Or with Bun:

```sh
bunx view-motion init
```

This scaffolds the core motion files into `src/motion/` (or `motion/` if you don't have a `src/` directory). You own these files — edit them freely.

The command also detects whether you have GSAP installed and prints the install command if not:

```sh
# If gsap is missing, init will print:
npm install gsap
# or pnpm add gsap / bun add gsap / yarn add gsap
```

### Install GSAP

If GSAP wasn't already in your project:

```sh
npm install gsap
```

### Move the page wrapper into your layout

This is the **most important structural requirement**. The `page-content` element must live in your root layout — not in individual pages — so the same DOM node persists across navigations. This is what gets cloned for the ghost exit animation.

```tsx filename="app/layout.tsx"
import { ViewMotionProvider } from '@/motion/provider'
import { motionConfig } from '@/motion/config'

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        <ViewMotionProvider config={motionConfig}>
          {/* Any fixed nav/header here */}
          <main className="page-content">
            {children}
          </main>
        </ViewMotionProvider>
      </body>
    </html>
  )
}
```

<Callout type="warning">
  If you put `<main className="page-content">` inside each individual page component instead of the layout, React will create a new DOM node on every navigation. The ghost clone and entry animation will both target the wrong element.
</Callout>

### Update your page components

Since the `page-content` wrapper is now in the layout, individual pages just return their content directly:

```tsx filename="app/page.tsx"
"use client"

export default function HomePage() {
  return (
    <div className="h-svh flex items-center justify-center">
      <h1>Hello world</h1>
    </div>
  )
}
```

### Add page entry animations (optional)

Use `usePageEntry` to animate elements in when a page loads:

```tsx filename="app/page.tsx"
"use client"

import { usePageEntry } from '@/motion'

export default function HomePage() {
  usePageEntry((tl) => {
    tl.from('.hero-title', { y: 40, opacity: 0, duration: 0.6 })
    tl.from('.hero-subtitle', { y: 20, opacity: 0, duration: 0.5 }, '<0.1')
  })

  return (
    <div className="h-svh flex flex-col items-center justify-center gap-4">
      <h1 className="hero-title text-6xl font-bold">Hello world</h1>
      <p className="hero-subtitle text-xl text-gray-400">Welcome to my site</p>
    </div>
  )
}
```

</Steps>

## Verifying it works

Start your dev server and navigate between pages using `<Link>` components. You should see:

1. The current page scales down and fades out (the ghost exit animation)
2. The new page clips in from the top simultaneously (the entry animation)

If the page stays invisible or pops in without animation, check that:
- `<main className="page-content">` is in `layout.tsx`, not in individual pages
- `<ViewMotionProvider>` wraps everything including the `page-content` element
- GSAP is installed

<Callout>
  The default transition is a clip-reveal from the top + scale-down exit. See [Transitions](/transitions/overview) to learn about built-in options and how to write your own.
</Callout>
