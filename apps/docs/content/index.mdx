# view-motion

**view-motion** is a page transition system for Next.js App Router, built on top of [GSAP](https://gsap.com). It handles the coordination between leaving and entering pages so you can focus on writing animations instead of managing lifecycle timing.

## How it works

When a user clicks a link, view-motion:

1. **Clones** the current page's content element into a fixed-position ghost overlay
2. **Runs your exit animation** on the ghost, while simultaneously navigating to the new route
3. **Hides and clips** the incoming page's content before the browser paints — no flash
4. **Runs your entry animation** (clip-in from the top, by default) as the new page mounts
5. **Removes the ghost** once the exit animation completes

Both animations run in parallel. The leaving page plays out while the entering page reveals beneath it.

## Key concepts

**Ghost clone** — rather than animating the real DOM node away (which would disappear when React unmounts it), view-motion clones the leaving page into a fixed overlay. The clone is purely visual; it has no event listeners and is removed when the exit animation finishes.

**`page-content` wrapper** — a single `<main className="page-content">` element in your layout wraps `{children}`. This is the element that gets cloned, clipped, and animated. Placing it in the layout (not in each individual page) ensures the same DOM node is reused across navigations.

**Scaffolded files** — unlike most animation libraries, view-motion copies its source files directly into your project via the CLI. You own the code and can modify it freely. This also means there is no runtime dependency to install separately beyond GSAP.

## Features

- Zero runtime configuration — works out of the box after `init`
- GSAP-powered animations with full access to GSAP's API
- Ghost clone approach for clean exit animations with no React lifecycle conflicts
- `usePageEntry` — register entry animations per page, scoped to that page's elements
- `usePageLeave` — register exit animations per page, scoped to the ghost clone
- Optional preloader with a built-in bar-wipe animation
- Namespace-based transition routing (different animations per route pair)
- Intercepting route support (Next.js parallel/modal routes)
- Automatically detects your package manager for install instructions
