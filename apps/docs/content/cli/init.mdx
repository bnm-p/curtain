# `view-motion init`

Scaffolds the core motion files into your project. You run this once when setting up view-motion for the first time.

```sh
view-motion init [options]
```

## What it scaffolds

The command copies the following files into your output directory (default: `src/motion/`):

| Destination | Purpose |
|---|---|
| `types.ts` | TypeScript types: `TransitionDef`, `MotionConfig`, `defineMotionConfig` |
| `context.tsx` | React context that coordinates animations, preloader state, and leave/entry registrations |
| `page-transition.tsx` | The core orchestrator: ghost clone, exit/entry timelines, click handler |
| `provider.tsx` | `<ViewMotionProvider>` — the component you add to your layout |
| `config.ts` | Your `motionConfig` — edit this to customise transitions and namespaces |
| `index.ts` | Barrel export for all public hooks and components |
| `hooks/use-page-entry.ts` | `usePageEntry` hook |
| `hooks/use-page-leave.ts` | `usePageLeave` hook |
| `transitions/default.ts` | The default clip-reveal + scale-out transition |

All files are copied as-is into your project. There is no compiled runtime — **you own the source**.

## Output directory detection

If your project has a `src/` directory, files go into `src/motion/`. Otherwise they go into `motion/`. You can override this with `--dir`.

## Options

| Flag | Type | Description |
|---|---|---|
| `-d, --dir <dir>` | `string` | Output directory, relative to cwd. Default: `src/motion` or `motion` |
| `--force` | `boolean` | Overwrite existing files. Without this flag, init exits if the output directory is non-empty. |

## After running init

**1. Install GSAP** (if not already installed):

The command detects your package manager and prints the exact install command:

```sh
# npm
npm install gsap

# pnpm
pnpm add gsap

# yarn
yarn add gsap

# bun
bun add gsap
```

**2. Add `ViewMotionProvider` to your layout:**

```tsx filename="app/layout.tsx"
import { ViewMotionProvider } from '@/motion/provider'
import { motionConfig } from '@/motion/config'

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        <ViewMotionProvider config={motionConfig}>
          <main className="page-content">
            {children}
          </main>
        </ViewMotionProvider>
      </body>
    </html>
  )
}
```

<Callout type="warning">
  The `<main className="page-content">` wrapper **must be in the layout**, not in individual pages. It is the element that gets cloned for the ghost exit animation and must be a stable DOM node across navigations.
</Callout>

**3. Edit `src/motion/config.ts`** to configure your transitions, namespaces, and timing. See [Configuration](/configuration) for the full reference.

## Examples

```sh
# Default — auto-detects output dir
npx view-motion init

# Custom output directory
npx view-motion init --dir app/motion

# Overwrite existing files
npx view-motion init --force

# Run with bun
bunx view-motion init
```

## Running locally in a monorepo

If you're working inside the view-motion repo itself and want to test the CLI against the example app:

```sh
# Build the CLI first
cd packages/cli && bun run build

# Run against the example app
cd apps/example && node ../../packages/cli/dist/view-motion.js init
```
