# `view-motion add`

Adds optional components to an existing `init`-ed project. Run this after `init` to extend your setup with a preloader or additional transition presets.

```sh
view-motion add <target> [options]
```

<Callout>
  `add` requires that you have already run `view-motion init`. It uses the same output directory detection as `init` to place files in the right location.
</Callout>

## Targets

### `preloader`

Adds a bar-wipe preloader overlay that plays on initial hard load. The preloader blocks the page entry animations until its animation completes, then wipes away to reveal the page.

```sh
view-motion add preloader
```

This copies `preloader.tsx` into your motion directory.

**Wire it up in your layout** using the `extras` prop on `<ViewMotionProvider>`:

```tsx filename="app/layout.tsx"
import { ViewMotionProvider } from '@/motion/provider'
import { motionConfig } from '@/motion/config'
import { Preloader } from '@/motion/preloader'

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        <ViewMotionProvider
          config={motionConfig}
          extras={<Preloader title="YOUR NAME" caption="Creative Developer" />}
        >
          <main className="page-content">
            {children}
          </main>
        </ViewMotionProvider>
      </body>
    </html>
  )
}
```

The `extras` prop renders outside `<PageTransition>` but inside the motion context. This is where the preloader (and any intercepting-route modals) should live.

See [Preloader](/preloader) for the full prop reference and customisation guide.

### `transition <name>`

Adds a named transition preset to your motion directory.

```sh
view-motion add transition <name>

# Shorthand — transition name directly:
view-motion add <name>
```

**Available transitions:**

| Name | Description | File |
|---|---|---|
| `default` | Clip-reveal from the top + scale-down exit. This is already included by `init`. | `transitions/default.ts` |
| `fade` | Simple cross-fade — old page fades out while new page fades in | `transitions/fade.ts` |

**Example:**

```sh
view-motion add transition fade
# or shorthand:
view-motion add fade
```

After adding, register the transition in `src/motion/config.ts`:

```ts filename="src/motion/config.ts"
import { fadeTransition } from './transitions/fade'
import { defaultTransition } from './transitions/default'
import { defineMotionConfig } from './types'

export const motionConfig = defineMotionConfig({
  // Use fade for all routes by default
  default: fadeTransition,

  // Or use it only for specific route pairs
  transitions: {
    'home -> about': fadeTransition,
    '* -> contact': fadeTransition,
  },
})
```

See [Transitions](/transitions/overview) for how namespaces and pair overrides work.

## Options

| Flag | Type | Description |
|---|---|---|
| `-d, --dir <dir>` | `string` | Output directory, relative to cwd. Default: auto-detected (`src/motion` or `motion`) |
| `--force` | `boolean` | Overwrite existing files. Without this flag, `add` exits if the target file already exists. |

## Examples

```sh
# Add the preloader
npx view-motion add preloader

# Add the fade transition
npx view-motion add transition fade
npx view-motion add fade     # shorthand

# Add to a custom directory
npx view-motion add preloader --dir app/motion

# Overwrite existing file
npx view-motion add preloader --force
```
