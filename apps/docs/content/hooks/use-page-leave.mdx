# `usePageLeave`

Registers element-level exit animations for the current page. When the user navigates away, these animations run on the **ghost clone** — the fixed-position snapshot of the leaving page — scoped so they only target elements within that clone.

```ts
import { usePageLeave } from '@/motion'
```

<Callout type="info">
  `usePageLeave` is fully optional. If a page doesn't call it, the ghost simply holds its current state while the transition's exit animation (scale-down, fade, etc.) runs on it. You only need `usePageLeave` when you want additional per-element animations during the exit.
</Callout>

---

## Signatures

### Timeline function (recommended)

```ts
usePageLeave((tl: gsap.core.Timeline) => void)
```

The callback receives a GSAP timeline. Tweens are added to `tl` and run on the ghost clone when the user navigates away. They run concurrently with the transition's exit animation.

```tsx
"use client"

import { usePageEntry } from '@/motion'
import { usePageLeave } from '@/motion'

export default function AboutPage() {
  usePageEntry((tl) => {
    tl.from('.heading', { y: 40, opacity: 0, duration: 0.6 })
  })

  usePageLeave((tl) => {
    tl.to('.heading', { y: -40, opacity: 0, duration: 0.5, ease: 'power2.in' })
  })

  return (
    <div>
      <h1 className="heading">About</h1>
    </div>
  )
}
```

### Free function

```ts
usePageLeave(() => void)
```

A zero-argument callback gives you full control. Note that with free functions, `gsap.context` scoping is not applied — manage your own selectors.

```tsx
usePageLeave(() => {
  gsap.to(document.querySelector('.heading'), {
    y: -40,
    opacity: 0,
    duration: 0.5,
  })
})
```

---

## Selector scoping

When you use the **timeline function** form, all GSAP string selectors are automatically scoped to the **ghost element** (the clone of the leaving page). This is critical — without scoping, `'.heading'` would match elements in both the ghost and the incoming page, causing both to animate unexpectedly.

```tsx
usePageLeave((tl) => {
  // Only targets .title inside the ghost clone, not in the incoming page
  tl.to('.title', { y: -40, opacity: 0, duration: 0.5 })
})
```

This scoping is applied via `gsap.context(fn, ghostElement)` internally.

<Callout type="warning">
  Free function callbacks (`() => void`) do **not** get scoped. If you query elements by class name in a free function, you may inadvertently target the incoming page's elements. Use `element.querySelector()` or `gsap.utils.toArray()` on the ghost element if you need free-function form.
</Callout>

---

## Relationship to the exit transition

The `usePageLeave` timeline runs **in parallel** with the `TransitionDef.exit()` animation. They are independent GSAP timelines both targeting the ghost.

The `TransitionDef.exit()` typically animates the ghost as a whole (e.g. `scale: 0.8, y: -30%, opacity: 0`). `usePageLeave` lets you add per-element animations on top of that — for example, staggering individual cards out before the whole ghost fades.

```tsx
usePageLeave((tl) => {
  // Stagger cards out — this runs alongside the ghost's scale-down exit
  tl.to('.card', {
    y: -20,
    opacity: 0,
    stagger: 0.05,
    duration: 0.4,
    ease: 'power2.in',
  })
})
```

---

## Cleanup on unmount

`usePageLeave` automatically clears its registered animation when the component unmounts. This prevents stale leave animations from running when navigating away from a page that previously registered one to a page that doesn't.

For example, if page A registers `usePageLeave` and page B doesn't, navigating from B to C will **not** run page A's leave animation. This is handled automatically.

---

## Coordinating entry and leave

A common pattern is to mirror your entry and leave animations so elements appear to move in one direction consistently:

```tsx
"use client"

import { usePageEntry, usePageLeave } from '@/motion'

export default function ProjectPage() {
  // Enter: slide up from below
  usePageEntry((tl) => {
    tl.from('.project-title', { y: 60, opacity: 0, duration: 0.6, ease: 'power3.out' })
    tl.from('.project-meta', { y: 30, opacity: 0, duration: 0.5 }, '<0.1')
    tl.from('.project-image', { scale: 1.04, opacity: 0, duration: 0.7 }, '<0.05')
  })

  // Leave: slide up and out (same direction → feels continuous)
  usePageLeave((tl) => {
    tl.to('.project-image', { scale: 0.97, opacity: 0, duration: 0.5, ease: 'power2.in' })
    tl.to('.project-meta', { y: -20, opacity: 0, duration: 0.4 }, '<')
    tl.to('.project-title', { y: -40, opacity: 0, duration: 0.45 }, '<0.05')
  })

  return (
    <article>
      <h1 className="project-title">My Project</h1>
      <p className="project-meta">2024</p>
      <img className="project-image" src="..." alt="" />
    </article>
  )
}
```

import type { TimelineAnimationFn } from '@/motion'
